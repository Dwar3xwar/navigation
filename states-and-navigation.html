States and Navigation

Let's stick with tradition and build a 'Hello World' example to start with. It will have two views that you can navigate between. The first view will display a Hyperlink that reads 'Hello' and the second will show a Hyperlink that says 'World'. Clicking the first Hyperlink takes you to second view and clicking the second Hyperlink takes you back to the first view.

We'll create a StateNavigator passing in an array of States that represent the views in our 'Hello World' example. So, there'll be one for the 'hello' view and another for the 'world' view. It's much too early to be thinking about routes, so we'll just indicate that the 'hello' view is the first to display by assigning it the empty route. 

var stateNavigator = new Navigation.StateNavigator([
    {key: 'hello', route: ''},
    {key: 'world'}
]);

To get notified when the 'hello' State is navigated to we'll attach a function to its navigated property. Whenever that State is navigated to our function will be called and we can then render the Hyperlink that makes up our 'hello' view. 

stateNavigator.states.hello.navigated = function() {
  // Render the 'hello' view
};

We'll use the NavigationLink component to build this Hyperlink that takes us to the 'world' State. This component has a stateKey property that holds the target State and another property that holds our stateNavigator. 

<NavigationReact.NavigationLink 
  stateKey="world"
  stateNavigator={stateNavigator}>
  Hello
</NavigationReact.NavigationLink>

Note

For times when we want to navigate without a Hyperlink, we can pass the target State into the navigate function on the StateNavigator, e.g., stateNavigator.navigate('world');
