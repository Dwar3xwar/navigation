Tutorial

This is an interactive tutorial about the Navigation routing library. It's a series of coding exercises, each one introducing a different Navigation topic. Edit the code and press the 'Run' button once you're happy. If you're right, a button will appear to take you to the next exercise. If you're wrong, an error will be written to the Console. Don't worry if you get stuck, pressing the 'Cheat' button reveals the answer and allows you to progress.

The Scenario

You've been tasked with building an application that allows users to edit a list of people. You're going to build it as a master details Single Page Application (SPA), consisting of two views. The first will display a paged list of people. Clicking on a person in the list will take the user to the second, an editable view of that person's details.

Exercise 1: Configuring States

Your SPA has two routes: a 'people' route for the listing view and a 'person' route for the details view. The Navigation library thinks in in terms of States rather than routes. There's a 1 to 1 relationship between a State and a route. In this first exercise, you'll create a listing State mapped to the 'people' route and a details State mapped to the 'person' route. You'll also wrap these States inside a Dialog. A Dialog groups related States togther, indicating which of them is the first to display.
