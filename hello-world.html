Tutorial

Hello World

Let's stick with tradition and build a 'Hello World' example to start with. It will have two views that you can navigate between. The first view will display a Hyperlink that reads 'Hello' and the second will show a Hyperlink that says 'World'. Clicking the first Hyperlink takes you to second view and clicking the second Hyperlink takes you back to the first view.

We'll create a StateNavigator passing in an array of States that represent the views in our 'Hello World' example. There'll be one for the 'hello' view and another for the 'world' view. It's much too early to be thinking about routes - we'll just assign the 'hello' State the empty route so it's displayed when the page first loads. 

var stateNavigator = new Navigation.StateNavigator([
  {key: 'hello', route: ''},
  {key: 'world'}
]);

To get notified when the 'hello' State is navigated to we'll attach a function to its navigated property. Whenever that State is navigated to our function will be called and we can then render the Hyperlink to take us to the 'world' State. We'll use the NavigationLink component to build this Hyperlink. This component has a stateKey property that holds the destination State and another property that holds our stateNavigator. 

stateNavigator.states.hello.navigated = function() {
  ReactDOM.render(
    <NavigationReact.NavigationLink 
      stateKey="world"
      stateNavigator={stateNavigator}>
      Hello
    </NavigationReact.NavigationLink>,
    document.getElementById('app'));
};

stateNavigator.states.world.navigated = function() {
  ReactDOM.render(
    <div>World</div>,
    document.getElementById('app'));
};

To get the example going we'll call start on the stateNavigator. This navigates to the State whose route matches the browser hash. The empty hash on initial page load means we'll see the 'hello' view first. 

stateNavigator.start();

Data First

stateNavigator.states.hello.navigated = function() {
  ReactDOM.render(
    <NavigationReact.NavigationLink 
      stateKey="world"
      navigationData={{size: 10}}
      stateNavigator={stateNavigator}>
      Hello
    </NavigationReact.NavigationLink>,
    document.getElementById('app'));
};
stateNavigator.states.world.navigated = function(data) {
  ReactDOM.render(
    <div style={{fontSize: data.size}}>World</div>,
    document.getElementById('app'));
};

Routes Last

It's not until the end of a project that you know what your routes look like. But, with other routers, you have to decide on them upfront and subsequent route changes can ripple through your code base. With the Navigation router, you can leave your route decisions until the end because changes don't affect your code base. 

var stateNavigator = new Navigation.StateNavigator([
  {key: 'hello', route: ''},
  {key: 'world', route: 'world/{size}', defaultTypes: {size: 'number'}}
]);
