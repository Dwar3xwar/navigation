State Navigation

Defining States

The Navigation router helps you navigate between views in your application. To get started, create a new StateNavigator passing in an array of States that represent the views in your application. Assign the empty route to the State you want to show when the application first loads. You can put off assigning the other routes until your application is up and running. In an email application we might have States representing the inbox and the views for reading and writing an email. 

var stateNavigator = new Navigation.StateNavigator([
  {key: 'inbox', route: ''},
  {key: 'mail'},
  {key: 'compose'}
]);

Rendering Views

The Navigation router doesn't render views for you because React natively supports loading components. Calling ReactDOM.render unmounts the old component and mounts the new component in its place. The Navigation router notifies you when a State becomes active and it's your job to render the relevant component for the active State. You receive the notification by attaching a function to the navigated property of the State. In the email example, we render our Inbox component into the 'app' placeholder when the 'inbox' State is active. 

stateNavigator.states.inbox.navigated = function() {
  ReactDOM.render(
    <Inbox />,
    document.getElementById('app'));
};

Navigating

States become active when you navigate to them. One way to navigate to a State is with a Hyperlink. The Navigation router encourages you to think about States rather than Urls. You can build Hyperlinks using the NavigationLink component, passing in the destination State via the stateKey prop. There's another prop for the stateNavigator, but you can pass it implicitly using the React context if you prefer. 

<NavigationReact.NavigationLink 
  stateKey="compose"
  stateNavigator={stateNavigator}>
  Compose Mail
</NavigationReact.NavigationLink>

When Hyperlinks aren't appropriate, you can navigate programmatically by passing the State key into the navigate function on the stateNavigator. 

stateNavigator.navigate('compose');

Calling start on the stateNavigator activates a State when the page loads. It triggers a navigation to the State whose route matches with the browser Url and ensures the State assigned the empty route is displayed first. 

stateNavigator.start();
