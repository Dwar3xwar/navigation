Navigation Knockout

Navigation Knockout is a plugin that lets you build Hyperlinks using Knockout's binding syntax. What follows is a series of code snippets that take you through the Knockout required for the different kinds of Hyperlinks.

Navigation to a Dialog

These snippets show how to Hyperlink to the inbox. Each begins with a comment recalling the equivalent navigation in JavaScript. 

<!-- Navigation.StateController.navigate('inbox'); -->
<a data-bind="navigationLink: 'inbox'"></a>

<!-- Navigation.StateController.navigate('inbox', { pageNumber: 2 }); -->
<a data-bind="navigationLink: 'inbox', toData: { pageNumber: 2 }"></a>

Navigation along a Transition

These snippets show how to Hyperlink from the inbox to the mail details. Each begins with a comment recalling the equivalent navigation in JavaScript. 

<!-- Navigation.StateController.navigate('select'); -->
<a data-bind="navigationLink: 'select'"></a>

<!-- Navigation.StateController.navigate('select', { id: 10 }); -->
<a data-bind="navigationLink: 'select', toData: { id: 10 }"></a>

Back Navigation

This snippet shows how to Hyperlink back to the inbox after viewing a mail. It begins with a comment recalling the equivalent navigation in JavaScript. 

<!-- Navigation.StateController.navigateBack(1); -->
<a data-bind="navigationBackLink: 1"></a>

Refresh Navigation

These snippets show how to Hyperlink to the second page of the inbox. Each begins with a comment recalling the equivalent navigation in JavaScript. 

<!-- Navigation.StateController.refresh({ pageNumber: 2 }); -->
<a data-bind="refreshLink: { pageNumber: 2 }"></a>


<!-- 
var data = Navigation.StateContext.includeCurrentData({ pageNumber: 2 }, ['sort','filter']);
Navigation.StateController.refresh(data); 
-->
<a data-bind="refreshLink: { pageNumber: 2 }, currentDataKeys: 'sort,filter'"></a>


<!-- 
var data = Navigation.StateContext.includeCurrentData({ pageNumber: 2 });
Navigation.StateController.refresh(data); 
-->
<a data-bind="refreshLink: { pageNumber: 2 }, includeCurrentData: true"></a>

Active

A Refresh Hyperlink is considered active if it's data is a subset of the current data. A Hyperlink to the second page of the inbox becomes active when the second page is viewed, regardless of any sort or filter criteria also in effect. A Dialog navigation Hyperlink is active if it matches the current Dialog and if it's data is a subset of the current data. Hyperlinks that navigate back or along a Transition can't be active. When a Hyperlink is active you can choose to disable it and/or add a css class name so it can be styled. 

<!-- Disable the link when active -->
<a data-bind="refreshLink: { pageNumber: 2 }, disableActive: true"></a>


<!-- Disable the link when active -->
<a data-bind="refreshLink: { pageNumber: 2 }, activeCssClass: 'active'"></a>

Navigation Handler

You can assign a handler to the navigating attribute of a Hyperlink. This handler function on your View Model will be called before the navigation takes place. 

<a data-bind="navigationLink: 'select', navigating: handler""></a>

The action you take inside your handler determines how the navigation behaves. 

// Cancel the Navigation and execute the Hyperlink
this.handler = function(event, url){
}

// Cancel the Navigation and the Hyperlink
this.handler = function(event, url){
    event.preventDefault();
}

// Execute the Navigation
this.handler = function(event, url){
    return true;
}

// Manually execute the Navigation
this.handler = function(event, url){
    event.preventDefault();
    Navigation.StateController.navigateLink(url);
}

Lazy

Hyperlinks are defined relative to the current State and data. A Hyperlink along a Transition will produce different hrefs whenever the current State changes. A Refresh Hyperlink with includeCurrentData set to true will produce different hrefs whenever the current data changes. So, when a navigation occurs, Hyperlinks auto update. If you have many 100s of Hyperlinks on a page this re-evaluation of the Hyperlinks can slow things down. Making a Hyperlink 'lazy' will defer this re-evaluation until it's clicked. Lazy Hyperlinks don't reliably report their active status. 

<a data-bind="refreshLink: { pageNumber: 2 }, lazy: true"></a>
