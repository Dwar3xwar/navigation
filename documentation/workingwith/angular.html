Navigation Angular

Navigation Angular is a plugin that lets you build Hyperlinks using Angular's binding syntax. What follows is a series of code snippets that take you through the Angular required for the different kinds of Hyperlinks.

Navigation to a Dialog

These snippets show how to Hyperlink to the inbox. Each begins with a comment recalling the equivalent navigation in JavaScript. 

<!-- Navigation.StateController.navigate('inbox'); -->
<a navigation-link="'inbox'"></a>

<!-- Navigation.StateController.navigate('inbox', { pageNumber: 2 }); -->
<a navigation-link="'inbox'" to-data="{ pageNumber: 2 }"></a>

Navigation along a Transition

These snippets show how to Hyperlink from the inbox to the mail details. Each begins with a comment recalling the equivalent navigation in JavaScript. 

<!-- Navigation.StateController.navigate('select'); -->
<a navigation-link="'select'"></a>

<!-- Navigation.StateController.navigate('select', { id: 10 }); -->
<a navigation-link="'select'" to-data="{ id: 10 }"></a>

Back Navigation

This snippet shows how to Hyperlink back to the inbox after viewing a mail. It begins with a comment recalling the equivalent navigation in JavaScript. 

<!-- Navigation.StateController.navigateBack(1); -->
<a navigation-back-link="1"></a>

Refresh Navigation

These snippets show how to Hyperlink to the second page of the inbox. Each begins with a comment recalling the equivalent navigation in JavaScript. 

<!-- Navigation.StateController.refresh({ pageNumber: 2 }); -->
<a refresh-link="{ pageNumber: 2 }"></a>


<!-- 
var data = Navigation.StateContext.includeCurrentData({ pageNumber: 2 }, ['sort','filter']);
Navigation.StateController.refresh(data); 
-->
<a refresh-link="{ pageNumber: 2 }" current-data-keys="'sort,filter'"></a>


<!-- 
var data = Navigation.StateContext.includeCurrentData({ pageNumber: 2 });
Navigation.StateController.refresh(data); 
-->
<a refresh-link="{ pageNumber: 2 }" include-current-data="true"></a>

Active

A Refresh Hyperlink is considered active if it's data is a subset of the current data. A Hyperlink to the second page of the inbox becomes active when the second page is viewed, regardless of any sort or filter criteria also in effect. A Dialog navigation Hyperlink is active if it matches the current Dialog and if it's data is a subset of the current data. Hyperlinks that navigate back or along a Transition can't be active. When a Hyperlink is active you can choose to disable it and/or add a css class name so it can be styled. 

<!-- Disable the link when active -->
<a refresh-link="{ pageNumber: 2 }" disable-active="true"></a>


<!-- Add a class name when active -->
<a refresh-link="{ pageNumber: 2 }" active-css-class="'active'"></a>

History Action

Clicking a Hyperlink updates the browser Url and creates a history entry. You can change this default behaviour by setting a history action. An action of 'Replace' changes the browser Url without creating a history entry, while an action of 'None' navigates without changing the browser Url. 

<!-- Replace the browser Url -->
<a navigation-link="'select'" history-action="'Replace'"></a>


<!-- Don't change the browser Url -->
<a refresh-link="{ pageNumber: 2 }" history-action="'None'"></a>

Navigation Handler

You can assign a handler to the navigating attribute of a Hyperlink. This handler function on your Controller will be called before the navigation takes place. 

<a navigation-link="'select'" navigating="handler($event, url)"></a>

The action you take inside your handler determines how the navigation behaves. 

// Cancel the Navigation and execute the Hyperlink
$scope.handler = function($event, url){
}

// Cancel the Navigation and the Hyperlink
$scope.handler = function($event, url){
    $event.preventDefault();
}

// Execute the Navigation
$scope.handler = function($event, url){
    return true;
}

// Manually execute the Navigation
$scope.handler = function($event, url){
    $event.preventDefault();
    Navigation.StateController.navigateLink(url);
}

Lazy

Hyperlinks are defined relative to the current State and data. A Hyperlink along a Transition will produce different hrefs whenever the current State changes. A Refresh Hyperlink with includeCurrentData set to true will produce different hrefs whenever the current data changes. So, when a navigation occurs, Hyperlinks auto update. If you have many 100's of Hyperlinks on a page this re-evaluation of the Hyperlinks can slow things down. Making a Hyperlink 'lazy' will defer this re-evaluation until it's clicked. Lazy Hyperlinks don't reliably report their active status. 

<a refresh-link="{ pageNumber: 2 }" lazy="true"></a>
