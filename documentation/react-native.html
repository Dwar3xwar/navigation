React Native

Like React, the Navigation router works on native apps as well as the web. The Navigation router provides a SceneNavigator component to handle the unique requirements of native app navigation, like populating and animating the stack. You can see it in action in the following Twitter example. When you select a tweet, the current view scales back as the new one slides in from the right. When you click the back arrow, the current view slides out to the right as the previous one scales back in.

Rendering Scenes

You create a new StateNavigator passing in an array of States that represent the scenes in your app, just like you would on the web. Make sure you set trackCrumbTrail true to enable the breadcrumb trail stack navigation. 

var stateNavigator = new StateNavigator([
  {key: 'home'},
  {key: 'tweet', trackCrumbTrail: true}
]);

Then you attach a renderScene function to each State. This is similar to the navigated function used on the web except that renderScene returns a component. 

stateNavigator.states.tweet.renderScene = function(data) {
  return <Tweet id={data.id} />;
}

Navigate around as you would programmatically on the web and the SceneNavigator renders the scene for the active State. There are four distinct stages a scene can pass through. You provide a style for each stage and the SceneNavigator animates the transitions between them. A scene starts off in the unmountedStyle; when a scene becomes active it changes into the mountStyle; when forward navigation moves it back in the stack it takes on the mountedStyle; and finally it's assigned the unmountStyle when back navigation removes it from the stack.

React Motion controls the animation so you specify the styles using its spring function. The animateStyle prop translates the interpolated values into a style object that's applied to the scene. Here's what the style props look like for the animation in the Twitter example: 

<SceneNavigator
  getUnmountedStyle={() => ({translate: 100, scale: 1})}
  getMountStyle={() => ({translate: spring(0), scale: spring(1)})}
  getMountedStyle={() => ({translate: spring(5), scale: spring(0.9)})}
  getUnmountStyle={() => ({translate: spring(100), scale: 1})}
  animateStyle={({translate, scale}) => ({
    transform: `translate(${translate}%) scale(${scale}, ${scale})`})}
  stateNavigator={stateNavigator} />

Notifying

An in-app notification takes a user directly to a particular scene, but if the stack isn't populated as expected then the back navigation becomes confusing. With the Navigation router the breadcrumb trail, or stack, is just a string, so you can populate the desired stack when jumping to a scene. You can test this out in the Twitter example: click a few tweets to build up a stack and then refresh the browser. You'll notice that the stack persists across refreshes because it's stored in the browser Url.

Note

The Navigation router for React Native hasn't been released yet. If you want to try it out take a look at the source code for the Twitter example.
