Tidying Up Urls

The Navigation router sees Urls as clothes draped over the body of your code, while other routers tattoo them into the skin. Clothes you can change without impacting the body, but tattoos you can't. The Navigation router ships with a scruffy set of clothes for your code to wear throughout development. You can dress it in some smarter togs once your appliction's running smoothly.

Let's take the example of an email application. The scruffy Url the Navigation router generates for opening an email with an id of '12' is 'mail?id=121_0'. We'll smarten up the Url in three stages so that, by the end, it will read 'open/12'. 

<NavigationReact.NavigationLink 
  stateKey="mail"
  navigationData={{id: 12}}
  stateNavigator={stateNavigator}>
  Open Mail
</NavigationReact.NavigationLink>

Configuring Routes

Because we've not supplied a route for the 'mail' State, the Navigation router uses the State's key instead. We'll change the Url from 'mail?id=121_0' to 'open?id=121_0' by setting the State's route property. 

var stateNavigator = new Navigation.StateNavigator([
  {key: 'inbox', route: ''},
  {key: 'mail', route: 'open'},
  {key: 'compose'}
]);

Note

You can assign an array to a State's route property and the Navigation router will pick the best match based on the navigation data.

Configuring Types

You'll recall that the Navigation router passes around typed data. By default it uses the Url to keep track of this type information. For example, The last three characters in the open mail Url track that the id is a number. We'll remove these trailing characters, so that the Url reads 'open?id=12' instead of 'open?id=121_0', by assigning the id a default type of 'number'. 

var stateNavigator = new Navigation.StateNavigator([
  {key: 'inbox', route: ''},
  {key: 'mail', route: 'open', defaultTypes: {id: 'number'}},
  {key: 'compose'}
]);

Note

  • You don't have to specify a default type for string values
  • You don't have to specify a default type if you've specified a default value
  • The supported default types are 'string', 'number, 'boolean', 'date', 'stringarray', 'numberarray', booleanarray' and 'datearray'.
  • If your string values can contain underscores then set trackTypes to false on your State, e.g., {key: 'mail', trackTypes: false}

Configuring Route Parameters

By default the Navigation router passes data in the query string. We'll change the Url from 'open?id=12' to 'open/12' by specifying the id as a route parameter. 

var stateNavigator = new Navigation.StateNavigator([
  {key: 'inbox', route: ''},
  {key: 'mail', route: 'open/{id}', defaultTypes: {id: 'number'}},
  {key: 'compose'}
]);

Note

  • You can indicate that a route parameter's optional with a '?' suffix, e.g., {id?}
  • You can map array values to splat parameters, indicated with a '*' prefix, e.g., {*ids} or {*ids?}
